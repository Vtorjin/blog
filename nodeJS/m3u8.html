# 油猴爬虫系列: 下载m3u8视频

最近爱奇艺会员到期了，想看的动漫看不了了，因此找了个替代的网站去白嫖免费的视频，然后发现该网站的视频m3u8(什么是m3u8?)地址在源码里面就可以找到, 因此决定用油猴和nodejs白嫖资源.

下列示例都以樱花动漫为准

# html文件分析

打开对应html后，按下F12在控制台里面的network选项可以看到以下信息:

network最先请求的是一个m3u8地址，所以我们就去html里面的源码查找看看是否能匹配到对应的地址.

  • 首先鼠标右键查看文件源码,会发现在文件的中部位置有一段文字记录了m3u8的地址所在.如图所示:

标记中心内容就是对应的请求地址，所以要做的就是将对应的段落提取出来。

  • 然后去看到刚刚获取的m3u8文件请求的内容，一般来说请求的文件内容一般包含以下了信息:
  1. 重定向m3u8位置信息. 比如

如果请求的文件还有 .m3u8的文本时表示当前请求的m3u8文件并不是真实的ts文件列表信息，因此我们需要对当前的地址行进行重新整合为真实的地址. 这种情况又会出现以下结果:

以xxx.m3u8为后缀,请求到新m3u8地址需要对'xxx.m3u8'进行替换，比如:

  origin:"https://vip.lz-cdn8.com/20220404/794_e47294f5/index.m3u8" //原始地址
  response:"1200k/hls/mixed.m3u8" //新地址文本
  targetUrl:"https://vip.lz-cdn8.com/20220404/794_e47294f5/1200k/hls/mixed.m3u8" //请求的真实地址

以https://www.xxx.com 为前缀,请求到的path地址需要和前缀进行拼接, 比如:

origin:"https://e1.monidai.com/20210411/iyZ7ss0x/index.m3u8",
response:"/ppvod/07A6BECA40DFAD5E3D425537679681F2.m3u8",
targetUrl:"https://e1.monidai.com/ppvod/07A6BECA40DFAD5E3D425537679681F2.m3u8"

  1. 纯ts文件列表信息(获取真实m3u8地址). 比如

    其中 #EXTINF:5.839表示该ts文件有5.839S的时长,axxxxxxxxxxx.ts,表示对应ts文件名,也是我们要请求的路径.

    在ts文件合并为mp4格式的文件时,m3u8的文件信息只能包含path.ts格式的文本,类似 https://test.com/1.ts或者 1.ts?auth=xxxxxxx这种格式的文本都会导致文件合并失败.因此需要确保m3u8的文本格式准确.

  2. 视频加密文件key
    比如#EXT-X-KEY:METHOD=AES-128,URI="/20220404/794_e47294f5/1200k/hls/key.key"或者key.key文本中有包含.key字样时.表示该视频文件是被加密了便无法直接打开.因此还需要请求对应的 .key文件，获取对应的文件内容.

# 前端提取有效信息

经过上面的分析,在本地环境下进行视频合并时,该功能大致需要确定以下几个要素:

  • html文件的m3u8地址提取
  • m3u8文件中是否存在重定向的地址
  • m3u8文件中是否存在key文件
  • m3u8文件中的ts格式需要进行格式处理
  • 如果存在key,key在m3u8中的内容也需要格式化处理
  • 生成带有请求ts文件地址的列表

为了测试功能是否可用,该脚本代码可以直接在本地写然后保存在油猴脚本中进行调试.为了直接获取dom元素，下面的都直接用Jquery代替.

# 提取html的m3u8地址

const crawler = {
    bodyHtml: '',   //获取去除空白字符后的文本信息;
    url: "",//m3u8地址
    rootUrl: "",//剔除.m3u8
    keyValue: "", //存放key文件的文本
    title: '', //视频名称可以从html标题中提取
    init() {
        this.bodyHtml = $('body').text().replace(/(\s*)|(\r)|(\n)/g, '');
        this.title = document.title.slice(0, document.title.indexOf("集") + 1);
        this.getUrlFromHtml();
    },
    // 从网页中获取数据
    getUrlFromHtml() {
        //** 该正则是根据对应网页的内容进行调整,不同的网站提取的地址不一样 **//
        const reg = /(?<=(,"url":"))(\S)*(?=(","url_next":|","next"))/g;
        const res = this.bodyHtml.match(reg);
        console.log(res);
        if (res && res[0]) {
            this.url = res[0].replaceAll(/\\/g, '');
            this.getRealUrl();
        }
    },

    //获取真实的m3u8地址
    getRealUrl() {
        let cacheUrl = this.url
        console.log(cacheUrl)
        fetch(cacheUrl)
            .then(t => t.text())
            .then(res => {
                let dataArr = res.split('\n');   // 用来收集的数据
                let hasMu = dataArr.find(t => t.includes('.m3u8'));  // 判断另外的m3u8地址
                //存在重定向的m3u8文件,对多个可能会出现的地址同时进行请求,请求成功的则为真实的地址 
                if (hasMu) {
                    Promise.allSettled([
                        this.replaceSuffix(cacheUrl, hasMu.trim()), 
                        this.replacePath(cacheUrl, hasMu.trim())
                        this.replaceFullUrl(hasMu)
                    ]).then(([rq1, rq2, rq3]) => {
                            console.log(rq1, rq2);
                    })
                } else {
                    // 不存在m3u8地址，对请求的内容进行格式化和收集地址的两步操作
                    this.reWriteMuFile(dataArr);
                }
            })
    },
    replaceSuffix(url = "", suffix = "index.m3u8") {
        var me = this;
        return new Promise((resolve, reject) => {
            url = url.split('/');
            suffix = suffix.split('/');
            let [firstTag, queryStr, idx] = [suffix[0], '', -1];  //后缀分割项的首个内容, 临时数据, 存在下标

            // case1: url:https://xx.cn/de/test/mix.m3u8   suffix: /test/index.m3u8;  
            // case2: url:https://xx.cn/de/test/mix.m3u8   suffix: test/index.m3u8;  
            queryStr = firstTag === '' ? suffix[1] : firstTag;
            // 根据新地址的第一项，判断原地址是否有可以替换的内容项
            idx = url.findIndex(str => str === queryStr);
            //  case A.1 url:https://xx.cn/de/test/mix.m3u8   suffix: 2000kb/hls/index.m3u8;
            if (idx === -1) {
                url.pop();
                url = `${url.join('/')}${firstTag == '' ? '' : '/'}${suffix.join('/')}`;
            } else {
                //  case A.2 url:https://xx.cn/de/test/mix.m3u8   suffix: test/2000kb/hls/index.m3u8;
                url = url.splice(0, idx);
                url = `${url.join('/')}${firstTag == '' ? '' : '/'}${suffix.join('/')}`;
            }

            $.ajax({
                url,
                success(res) {
                    console.log(res)
                    me.url = url;
                    me.reWriteMuFile(res.split('/n'));
                    resolve(true);
                },
                error() {
                    reject(false);
                }
            })
        })
    },
    
    // case: origin: https://A.cn/normal/index.m3u8 response: https://A.cn/other/index.m3u8
    replacePath(url = "", path) {
        var me = this;
        return new Promise((resolve, reject) => {
            let host = url.split("/").splice(0, 3).join('/');
            url = `${host}/${path[0] == "/" ? path.replace('/', '') : path}`;
            $.ajax({
                url,
                success(res) {
                    console.log(res)
                    me.url = url;
                    me.reWriteMuFile(res.split('/n'));
                    resolve(true);
                },
                error() {
                    reject(false);
                }
            })
        })
    },

    // case: origin: https://A.cn/index.m3u8 response: https://B.cn/index.m3u8
    replaceFullUrl(url) {
        var me = this;
        return new Promise((resolve, reject) => {
            url = url.match(/http(\S)*m3u8/gi);
            if (!url) return reject();
            url = url.shift();
            $.ajax({
                url,
                success(res) {
                    // console.log(res)
                    me.url = url;
                    me.rootUrl = url.slice(0, url.lastIndexOf('/'));
                    me.reWriteMuFile(res.split('\n'));
                    resolve(true);
                },
                error() {
                    reject(false);
                }
            })
        })
    },

    reWriteMuFile() {
        // !todo
    }
}

从html代码中提取有效的m3u8地址后，判断真实的请求地址，然后对请求的内容再次进行请求，这里就可以获得key文本,以及整个m3u8的请求ts文件,下一步可以进行请求key文件,格式化m3u8文件以及生成请求ts的list.

# 格式化m3u8中的ts内容

const crawler = {
    m3u8Txt: "",
    tsList:[], //ts的请求列表
    reWriteMuFile(dataArr = []) {
        var me = this;
        me.m3u8Txt = dataArr.map(line => {
            return line.includes('.ts') ? me.formatTsName(line, me.tsList, me.rootUrl) : !line.includes('.key') ? line : me.formatKeyName(line);
        }).join('\n');
    },

    // 格式化ts文本
    formatTsName(tsline = "https://test.com/2000kb/hls/8AqdvbKb.ts?auth=admin", container = [], rootUrl = "") {
        var ts = tsline.split('/').pop();
        //通过地址传进来的容器收集ts请求列表
        tsline.includes('http') ? container.push(tsline) : rootUrl && container.push(`${rootUrl}/${ts}`);
        return !ts.includes('?') ? ts : ts.slice(0, ts.indexOf('?'));
    },
    
    // 格式化key文本
    formatKeyName(line) {
        var keyUrl = line.match(/(?<=(URI="))(\S)*.key/g);
        this.getEncryptKey(keyUrl[0], this.rootUrl)
        return keyUrl ? line.replace(keyUrl[0], 'key.key') : line;
    }
}

# 下载对应的文件

const crawler = {
    downloadTxtFile(fileName, content) {
        // 它适用于所有支持 HTML5 的浏览器，因为它使用了 <a> 元素的下载属性：
        let element = document.createElement("a");
        // Blob 是一种可以存储二进制数据的数据类型
        // 根据要保存的文件，它可以有不同的值
        const blob = new Blob([content], { type: "plain/text" });
        // createObjectURL() 静态方法创建一个 DOMString，其中包含一个 URL，该 URL 表示参数中给定的对象。
        const fileUrl = URL.createObjectURL(blob);
        // setAttribute() 设置指定元素的属性值。
        element.setAttribute("href", fileUrl); // 文件位置
        element.setAttribute("download", fileName); // 文件名
        element.style.display = "none";
        // 使用 appendChild() 方法将一个节点附加到指定父节点的子节点列表的末尾处
        document.body.appendChild(element);
        element.click();
        // Node 接口的 removeChild() 方法从 DOM 中移除一个子节点并返回移除的节点
        document.body.removeChild(element);
    },
    downLoadAll() {
        var me = this;
        // 判断是否有key文件
        this.keyValue ? Promise.all([
            me.downloadTxtFile(`${me.title}.key`, me.keyValue),
            me.downloadTxtFile(`${me.title}.m3m8`, me.m3u8Txt),
            me.downloadTxtFile(`${me.title}.txt`, me.tsList.join("\n"))
        ]) : Promise.all([
            me.downloadTxtFile(`${me.title}.m3m8`, me.m3u8Txt),
            me.downloadTxtFile(`${me.title}.txt`, me.tsList.join("\n"))
        ])
    },
}

# NodeJs开启下载

在nodeJS中,ffmpeg有多种方式可以下载m3u8格式源的视频,但是在某些特殊情况下(比如网络延迟),下载的视频可能会出现某些时间段视频资源不完整,并且视频是全部下载,往往下载时间会稍微久的问题。通过ts列表下载的方式可以尽量保证视频资源的完整性,并且还可以自定义视频的开始结束时间,或者对视频的范围进行指定.

在ts文件合并为mp4格式时,一般包含由包含视频名的文件夹, m3u8文件和多个ts文件组成.在前面已经获取到对应的ts路径请求列表和本地的m3u8文件，接下来要做的就是将他们放置在同一个文件夹里面,然后进行请求ts文件的操作。

  • # 首先,根据videoName.m3u8文件的内容创建相关文件夹

const fs = require('fs');
const path = require('path');

// 创建文件夹
function createDir(myPath) {
    !fs.existsSync(myPath) && mkDirs(myPath);
}

function mkDirs(dirPath) {
    if (!fs.existsSync(path.dirname(dirPath))) {
        mkDirs(path.dirname(dirPath));
    }
    fs.mkdirSync(dirPath);
}

// 获取当前文件在node项目中的位置
function getPath() {
    return path.join(__dirname, ...arguments);
}

// 判断是否有文件或者文件夹
function hasFile(folderName) {
    return fs.existsSync(folderName);
}

// 移动原有的文件到新的文件夹中并重新命名
function moveFile(oldPath, newPath) {
    !!fs.existsSync(oldPath) && fs.renameSync(oldPath, newPath);
}


// 生成文件夹并获取ts列表,视频名称等相关数据
function createMuFileToPath(muName = "index.m3u8", oldFolder = "source", newFolder = "video") {
    const videoName = muName.includes('.m3u8') ? muName.split('.')[0] : muName; //视频名称
    const folder = getPath(newFolder, videoName.split('(')[0].replace(/(\s*)|(\r)|(\n)/g, '')); //新的文件夹路径
    const origin = getPath(oldFolder, muName); //原m3u8文件存放的路径 
    const destination = path.join(folder, 'index.m3u8');//新的m3u8文件存放的路径
    let [data, oriDataPath, desDataPath, size] = [[], "", "", 0]; //请求列表, 原始数据path, 目标数据path,
    // 1st: 创建目标文件夹
    createDir(folder);
    // 2nd: 读取txt文件中的ts列表数据
    try {
        // 如果用于目标文件夹中没有读取到对应的file,说明未创建
        if (!hasFile(destination)) {
            oriDataPath = origin.replace('.m3u8', '.txt');
            data = fs.readFileSync(oriDataPath, 'utf-8').split("\n"); //列表数据从原txt文件中读取
            desDataPath = destination.replace('.m3u8', '.txt');
            size = data.length;
            // 3rd: 移动m3u8, txt, key文件, 并统一命名为index.txt,index.m3u8,key.key;
            Promise.all([moveFile(origin, destination), moveFile(oriDataPath, desDataPath), moveFile(origin.replace('.m3u8', '.key'), path.join(folder, 'key.key'))])
        } else {
            // 已经创建了的文件地址
            desDataPath = destination.replace('.m3u8', '.txt');
            let downloadedTs = fs.readdirSync(folder).filter(ts => ts.includes('.ts'));
            // console.log(downloadedTs, 'yixiazai', folder)
            let originData = fs.readFileSync(desDataPath, 'utf-8').split("\n");
            data = originData.filter(ts => {
                return !downloadedTs.includes(ts);
            }); //列表数据从新txt文件中读取还未下载的内容
            size = originData.length; //总下载量
        }
    } catch (e) {}
    return {
        data,
        destination,
        folder,
        videoName,
        size
    };
}

module.exports = {
    createMuFileToPath,

/**
 * @fileName requestTS.js

const { existsSync, createWriteStream } = require('fs');
const { join } = require('path');
const request = require('request');
module.exports = function (tsList = [], folderName = "") {
    return new Promise((resolve, reject) => {
        const requestTS = () => {
            let times = 0;
            try {
                if (!tsList.length) {

                    resolve();
                    return
                };
                const url = tsList.shift();
                const tsName = url ? url.split('/').pop() : "";
                // 剔除请求参数
                const tsFile = ts = join(folderName, tsName.slice(0, tsName.indexOf('.ts') + 3));
                if (existsSync(tsFile)) {
                    requestTS();
                    return;
                }
                request({
                    url,
                    timeout: 5000
                }, function (err, response) {
                    if (!err && response.statusCode == 200) {
                        times = 0;
                        return;
                    }
                    times++;
                    if (times % 6 == 0) {
                        setTimeout(() => { requestTS() }, 1000);
                    } else {
                        tsList.unshift(url);
                        setTimeout(() => { requestTS() }, 1000);
                    }
                }).pipe(createWriteStream(tsFile))
                    .on('finish', () => { setTimeout(() => { requestTS() }, 300) })
            } catch (e) {
                reject()
            }
        }
        requestTS();
    })
}

# ffmpeg合并mp4

ts文件合并为mp4有很多种方式可以实现,这里我们采取视频工具ffmpeg进行处理,相关安装配置可参考(这里). 在ts列表资源请求结束后,合并期间会阻塞后续代码执行,可以开启新的进程进行视频合并的操作.

/**
 * @FileName createFile.js
 * */ 
const fs = require('fs');
const path = require('path');
const child_process = require('child_process').exec;

function mereVideo(folder, videoName) {
    let name = `${videoName}.mp4`;
    return new Promise((resolve,reject)=>{
        // 判断视频是否已经存在
        fs.access(path.join(folder, name), (e) => {
            if (!e) {  return resolve() };
            const code = `cd "${folder}" &&  ffmpeg -allowed_extensions ALL -i index.m3u8 -c copy "${name}"`
            child_process(code, function (err) {
                if(err){
                    console.log('视频合并失败!')
                    reject();
                }else{
                    console.log('进行下一个视频资源请求的操作!');
                    resolve();
                }
            });
        })
    })
}

/**
 *@FileName createFile.js
 * */ 
function mergeVideo(folder, videoName) {
    const name = `${videoName}.mp4`;
    const code = `cd "${folder}" &&  ffmpeg -allowed_extensions ALL -i index.m3u8 -c copy "${name}"`
    const videoPath = path.join(folder, name);
    fs.existsSync(videoPath) && fs.unlinkSync(videoPath); // 删除之前合并的视频
    return new Promise((resolve, reject) => {
        recurMerge(code, { f: folder, p: videoPath }, () => { return resolve() }, () => { return reject() })
    })
}

function recurMerge(code = "", position = { f: "", p: "" }, sucCallBack, errCallBack, isTwice = false) {
    child_process(code, function (err) {
        if (!err) {
            sucCallBack = sucCallBack ? sucCallBack : () => { }
            errCallBack = errCallBack ? errCallBack : () => { }
            return isTwice ? errCallBack() : sucCallBack();
        }
        fs.existsSync(position.p) && fs.unlinkSync(position.p);  // 删除已经合并的不完整视频

        let mergeInfo = [...arguments].filter(t => {
            return typeof t === 'string' && t.includes('.ts')
        }).pop();

        // 提取ts列表信息
        mergeInfo = mergeInfo.split("\n").filter(line => line.includes('.ts'));
        try {
            // errItem为倒数第二个ts，没有文件就清除最后5个ts
            let [lastItem, errItem, tsName] = [mergeInfo.pop(), mergeInfo.pop(), '']
            tsName = !!errItem ? errItem.match(/'\S*'/g) : lastItem.match(/'\S*'/g);
            let errPath = path.join(position.f, tsName[0].replace(/'/g, ''));
            fs.unlinkSync(errPath);
            setTimeout(() => {
                recurMerge(code, position, sucCallBack, errCallBack, true);
            }, 100)
        } catch (e) {
            // 删除最后5个ts
            mergeInfo = mergeInfo.length > 5 ? mergeInfo.splice(mergeInfo.length - 5) : mergeInfo.splice(0);
            while (mergeInfo.length) {
                let info = mergeInfo.pop();
                let tsName = info && info.match(/'\S*'/g);
                if (tsName && tsName[0]) {
                    let errPath = path.join(position.f, tsName[0].replace(/'/g, ''));
                    fs.existsSync(errPath) && fs.unlinkSync(errPath);
                }
            }
            return errCallBack ? errCallBack() : () => { }
        }
    })
}

# 开启分线程提速

上面的代码包含了请求m3u8所涵盖的ts资源,并最终合并为mp4格式文件的基本流程。但是该过程里面有许多可以优化的地方,比如:

  • 当前的时刻只能进行一次ts资源请求,速度慢.
  • 请求失败的文件无法自动重新获取

# 如何实现提速

要实现分线程加速的功能,可以用到worker-threads这个包中的Worker. 在主线程和分线程之间可以用postMessage,workData进行通信.在每一个线程中可以平均分配要请求的资源数量. 线程开启的数量取决于电脑运行内存和网络条件,两个条件越好支持的数量越多.开启加速后需要对上述的代码进行修改.

# 失败文件重新获取

由于网络原因，在request请求资源失败后,会自动在文件夹中创建空的ts文件，这会让系统判断该文件已经被下载成功了,并可能会导致最后视频合并失败.因此将本轮请求的文件进行错误文件过滤,过滤结束后,如果存在错误文件就进行重新本轮的请求,没有错误文件就进行视频合并和进行下一个请求任务. 这里对本轮重复次数就不做限制了,可以自己进行修改。

# 防止进程崩溃

通过上面的代码进行爬取时，由于我这里的网速不是很好，经常会出现程序自动退出的问题,除了网速不好, 正常情况下电脑的运行内存也是一个因素,当线程开的过多时,也会导致服务崩溃。为了解决这个问题,在nodejs官网找到了如下方案:

/**
 * @FileName processManager.js
 **/ 
const fs = require('fs-extra');
const { join } = require('path');
const child_process = require('child_process').fork;
let start;

function runTask() {
    // 相当于执行 node app.js的启动命令
    start = child_process(join(__dirname, './app.js'));
    start.on('exit', function (code) {
        try {
            start.kill();
            start = null;
            // console.log('已经杀掉!')
        } catch (e) {
            // console.log('重启试试')
        }
        // 程序退出后自动重启
        if (code == 0) {
            runTask();
            // console.log('尝试了重启了')
        }
    })
}
runTask();
