# ECMAScript历代版本特性 - ES13(2022)

2022 年 6 月 22 日，第 123 届 Ecma 大会批准了 ECMAScript 2022 语言规范[1]，这意味着它现在正式成为标准。

# New members of classes

# 简易变量声明

class 支持在 constructor 外部直接声明变量，过去我们必须在内部声明。

class Cat {
  gender = 'female'
}
let cat = new Cat()
cat.gender // female

# 私有变量和私有方法

封装是面向对象编程的核心原则之一。它通常使用可见性修饰符来实现，例如 private or public class 的最近改动里加入了 #，用来将类中的变量、方法或访问器标记为私有，从外部访问私有化的内容，会直接报错。

class MyClass {
  instancePublicField = 1; //实例公有属性
  static staticPublicField = 2; //静态公有属性

  #instancePrivateField = 3; //实例私有属性
  static #staticPrivateField = 4; //静态私有属性

  #nonStaticPrivateMethod() {} //非静态私有方法
  get #nonStaticPrivateAccessor() {}
  set #nonStaticPrivateAccessor(value) {}

  static #staticPrivateMethod() {} //静态私有方法
  static get #staticPrivateAccessor() {}
  static set #staticPrivateAccessor(value) {}

  static {
    // 静态初始化代码块
  }
}

# in 操作符检查私有属性

这个操作也被成为 “人性化检查私有属性的方式”. 后面的表达式就是示例 –他检查 obj 是否有一个私有属性 #privateSlot:

class ClassWithPrivateSlot {
  #privateSlot = true;
  static hasPrivateSlot(obj) {
    return #privateSlot in obj;
  }
}

const obj1 = new ClassWithPrivateSlot();
ClassWithPrivateSlot.hasPrivateSlot(obj1); //true

const obj2 = {};
ClassWithPrivateSlot.hasPrivateSlot(obj2);  //false

# Top-level await

我们现在可以在模块的顶层使用 await 并且不再需要输入异步函数或方法
应用场景:

  • 资源初始化
// my-module.js
const MongooseConnector = () => {
    // connect mongodb code
}

const dbConnection = await MongooseConnector();
  • 动态加载模块
const strings = await import(`/i18n/${navigator.language}`);
  • 依赖回退 依赖发生故障时，可以回退到旧版本，防止程序崩溃。
let jQuery;
try {
  jQuery = await import('https://cdn-a.com/jQuery');
} catch(e) {
  jQuery = await import('https://cdn-b.com/jQuery');
}

# Error.cause

Error及其子类现在可以让我们指定哪个错误导致了当前错误：

async function doJob() {
  const rawResource = await fetch('/test')
    .catch(err => {
      throw new Error('下载资源失败', { cause: err });
    });
}

try {
  await doJob();
} catch (e) {
  console.log(e);
  console.log('Caused by', e.cause);
}
// Error: 下载资源失败
// Caused by TypeError: Failed to fetch

# .at()

支持数据类型: Array,String .at() 让我们可以读取给定索引处的元素 (就像 []) 而且支持负数 (与 []不同):

// 数组及数组元素
const cart = ['apple', 'banana', 'pear'];
// 一个函数，用于返回给定数组的最后一个项目
function returnLast(arr) {
  return arr.at(-1);
}

cart.at(1); // 'banana'
returnLast(cart); // 'pear'

// 字符串
const someString =  'this is a word for test';
function getLastStr(str = ""){
    return str.at(-1)
}

someString.at(12); // 'r'
getLastStr(someString); //'t'

# RegExp match indices

修饰符除了g、i，s,正则表达式新增d标志会生成匹配对象，记录每个组捕获的开始和结束。
有不同的方法来匹配索引

  • 编号组的匹配索引
//  /d Modifier
const matchObj = /(a+)(b+)/d.exec('aaaabb');
console.log(matchObj); //  ['aaaabb', 'aaaa', 'bb', index: 0, input: 'aaaabb', groups: undefined, indices: Array(3)]

//  /s Modifier
console.log(/(a+)(b+)/s.exec('aaaabb')); //['aaaabb', 'aaaa', 'bb', index: 0, input: 'aaaabb', groups: undefined]

可以看到matchObj 还有一个属性 .indices 记录输入字符串中捕获的每个编号组。

.indices: Array(3)  
0: (2) [0, 6]
1: (2) [0, 4]
2: (2) [4, 6]
  • 命名组的匹配索引
const matchObj = /(?<as>a+)(?<bs>b+)/d.exec('aaaabb');

console.log(matchObj);
/*
Output - 
['aaaabb', 'aaaa', 'bb', index: 0, input: 'aaaabb', groups: {as: 'aaaa', bs: 'bb'}, indices: Array(3)]
*/

它们的索引存储在matchObj.indices.groups

indices: { as: [0,4], bs: [4,6] }

# Object.hasOwn(obj, propKey)

这是检查obj对象自身属性propKey的一种安全方法。它类似于Object.prototype.hasOwnProperty但它支持所有对象类型。

const proto ={
    protoProp:"pooo"
}
 
const obj = {
    __proto__:proto,
    objProp:"objProp"
}

console.log('protoProp' in proto) // true
Object.hasOwn(obj,'protoProp') // false
Object.hasOwn(proto,'protoProp') // false
