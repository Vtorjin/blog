# ECMAScript历代版本特性 - ES10(2019)

# String.prototype.trimStart()

该方法会删除一段字符串最前面的空白字符,trimLeft是该方法的别名.

// 从当前字符串删除数组中指定的一组字符的所有前导匹配项。
const greeting = '   Hello world!   ';
greeting.trimStart(); //删除前面的空白字符;  'Hello world!   '

# String.prototype.trimEnd()

和trimStart的删除位置相反,同时trimRight是该方法的别名.

const greeting = '   Hello world!   ';
greeting.trimEnd(); //'   Hello world!'

# Object.fromEntries()

Object.fromEntries()方法是Object.entries()的逆操作，用于将一个键值对数组转为对象

Object.fromEntries([
  ['foo', 'bar'],
  ['baz', 42]
])
// { foo: "bar", baz: 42 }

适用场景:

  • 是将键值对的数据结构还原为对象，因此特别适合将Map结构转为对象。
// 例一
const entries = new Map([
  ['foo', 'bar'],
  ['baz', 42]
]);

Object.fromEntries(entries)

// 2
const map = new Map().set('foo', true).set('bar', false);
Object.fromEntries(map)
// { foo: true, bar: false }
  • 配合URLSearchParams对象，将查询字符串转为对象。
location.href = 'https://cn.bing.com/search?q=object.fromEntries&fo…369D12D84936A36E40C5545EBE7C&ghsh=0&ghacc=0&ghpl=';
Object.fromEntries(new URLSearchParams(location.search))  // {"q":"object.fromEntries","form":"CHRDEF","sp":"-1","pq":"object.fromentries","sc":"8-18","qs":"n","sk":"","cvid":"B080369D12D84936A36E40C5545EBE7C","ghsh":"0","ghacc":"0","ghpl":""}

# Array.prototype.flat()

用法: Array.prototype.flat(number) 将多维数组进行降维操作，number为数字, 默认值为1,表示当前数组级别降低一级, number为Infinity时,结果最终为一维数组.

[1, 2, [3, 4]].flat();           // [ 1, 2, 3, 4 ]
[1, 2, [3, 4, [5, 6]]].flat(2);  // [ 1, 2, 3, 4, 5, 6 ]
[1, 2, [3, 4, [5, 6,['third']]]].flat(2); //[1, 2, 3, 4, 5, 6, ['third']]
 [1, 2, 3, 4, 5, 6, ['third']].flat(Infinity); // [1, 2, 3, 4, 5, 6, 'third']

# Array.prototype.flatMap()

用法: array.flatMap()

对嵌套数组中的数据可以再次进行遍历操作,首先使用映射函数映射每个元素，然后将结果压缩成一个新的数组，其中每个元素都是回调函数的结果，并且结构深度 depth 值为 1。

const arr1 = [1, 2, [3], [4, 5], 6, []].flatMap(num => num); // [1, 2, 3, 4, 5, 6]
const arr2 = [1, 2, [32,4,53], [1,3,2,['2ndArr']]].flatMap(num => num);  //  [1, 2, 32, 4, 53, 1, 3, 2, Array(1)] 
const arr3 = [1, 2, [32,4,53], [1,3,2,['2ndArr']]].flatMap(num => Array.isArray(num) ? num.flatMap(t => t ) : num); // [1, 2, 32, 4, 53, 1, 3, 2, '2ndArr']
const arr4 = [1,2,3,[4,5,6],[7,8,9]].flatMap(num => Array.isArray(num) ? num.flatMap(t => t * 2 ) : num * 2);

# Try Catch

2019版规定try{}catch(e){}错误信息(e)可选 2021版本又规定必选

// before 2019
try {
  // some code
}
catch (err) {
  // error handling code
}

// 2019
try  {
  // some code
}
catch {
  // error handling code
}

# JSON.stringify() 加强格式转化

JSON.stringify在 ES10 修复了对于一些超出范围的 Unicode展示错误的问题。因为 JSON 都是被编码成UTF-8，所以遇到 0xD800–0xDFFF 之内的字符会因为无法编码成 UTF-8 进而导致显示错误。在 ES10 它会用转义字符的方式来处理这部分字符而非编码的方式，这样就会正常显示了。

console.log(JSON.stringify('\uD83D\uDE0E')) // "😎"
JSON.stringify('\uD83D\uDE0F') //'"😏"'

// 如果我们只去其中的一部分  \uD83D 这其实是个无效的字符串
// 之前的版本 ，这些字符将替换为特殊字符，而现在将未配对的代理代码点表示为JSON转义序列
console.log(JSON.stringify('\uD83D')) // "\ud83d"

# Symbol.prototype.description

描述属性是只读的，可用于获取符号对象的描述，更好了解它的作用。Symbol 对象可以通过一个可选的描述创建，可用于调试，但不能用于访问 symbol 本身。Symbol.prototype.description属性可以用于读取该描述。与 Symbol.prototype.toString() 不同的是它不会包含"Symbol()"的字符串。

// toString
const name = Symbol('es')
console.log(name.toString()) // Symbol(es)

// description
Symbol('desc').description999999999+;
Symbol('desc').toString();   // "Symbol(desc)"
Symbol('desc').description;  // "desc"
Symbol('').description;      // ""
Symbol().description;        // undefined

// well-known symbols
Symbol.iterator.toString();  // "Symbol(Symbol.iterator)"
Symbol.iterator.description; // "Symbol.iterator"

// global symbols
Symbol.for('foo').toString();  // "Symbol(foo)"
Symbol.for('foo').description; // "foo"

# Array.prototype.sort() 排序更加稳定

早先的 ECMAScript 没有规定，**Array.prototype.sort()**的默认排序算法是否稳定，留给浏览器自己决定，这导致某些实现是不稳定的。ES2019 明确规定，**Array.prototype.sort()**的默认排序算法必须稳定

const arr = ["peach", "straw", "apple", "spork"];

const stableSorting = (s1, s2) => {
  if (s1[0] < s2[0]) return -1;
  return 1;
};

arr.sort(stableSorting);
